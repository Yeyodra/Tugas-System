name: template-creator
description: >
  Creates new task-type YAML files from natural language description.
  User describes the assignment format, this layer parses it into a valid
  YAML following the task-type schema, and saves to templates/task-types/custom/.
---

# Purpose

Enable users to define new assignment types without editing code. User describes the format in natural language (Indonesian or English), and this layer produces a valid task-type YAML that `content-generator.md` can consume.

# Input

| Parameter | Required | Description |
|-----------|----------|-------------|
| `description` | YES | Natural language description of the assignment format |
| `task_name` | NO | Desired YAML filename (auto-generated from description if not provided) |
| `matkul` | NO | Course name (for context-aware defaults) |
| `example_file` | NO | Path to an existing assignment for structure reference |

# Output

- New YAML file at `templates/task-types/custom/{task_name}.yaml`
- Confirmation summary showing the parsed structure
- Ready to use immediately with `content-generator.md`

---

# Process

## Step 1: PARSE — Extract Structure from Description

### 1.1 Identify Sections

Parse the user's description to identify distinct sections. Look for:

- **Explicit lists**: "formatnya: cover, pendahuluan, isi, kesimpulan"
- **Numbered items**: "1. Cover 2. Latar belakang 3. Pembahasan"
- **Comma-separated**: "cover, identitas paper, ringkasan, critical review"
- **Paragraph description**: "tugas ini harus ada cover dulu, terus isinya tentang analisis..."

**Section name normalization:**

| User Input (various forms) | Normalized ID | Default Title |
|---------------------------|---------------|---------------|
| cover, halaman depan, sampul | `cover` | (cover page, not a section) |
| pendahuluan, latar belakang, background | `latar_belakang` | Latar Belakang |
| identitas paper, info paper, paper identity | `identitas_paper` | Identitas Paper |
| ringkasan, summary, rangkuman | `ringkasan` | Ringkasan |
| metode, metodologi, methodology | `metode` | Metode Penelitian |
| pembahasan, analisis, discussion, analysis | `pembahasan` | Pembahasan |
| review, critical review, ulasan kritis | `critical_review` | Critical Review |
| kode, source code, listing program | `source_code` | Source Code |
| logika, penjelasan logika, logic explanation | `logika` | Penjelasan Logika Program |
| hasil, results, output | `hasil` | Hasil |
| kesimpulan, conclusion, penutup | `kesimpulan` | Kesimpulan |
| saran, recommendations | `saran` | Saran |
| daftar pustaka, referensi, bibliography, references | `daftar_pustaka` | Daftar Pustaka |
| lampiran, appendix | `lampiran` | Lampiran |

### 1.2 Infer Section Types

For each identified section, infer the appropriate `type`:

| Section Pattern | Inferred Type | Reasoning |
|----------------|---------------|-----------|
| Cover / halaman depan | (cover config) | Always `auto` — filled from student data |
| Identitas paper / info paper | `auto` | Filled from journal metadata |
| Source code / kode program | `code_listing` | Read from file |
| Logika / penjelasan logika | `delegate` → `code-analyzer` | Needs code analysis |
| Daftar pustaka / referensi | `reference` | Delegate to reference-finder |
| Kesimpulan pribadi / pendapat | `user_input` | Student's own opinion |
| Everything else | `generate` | AI-generated content |

### 1.3 Detect Special Requirements

Look for keywords indicating special handling:

| Keyword | Action |
|---------|--------|
| "APA", "daftar pustaka", "referensi" | Enable bibliography |
| "kode", "program", "source code" | Add code_listing section |
| "jurnal", "paper", "artikel" | May need journal-analyzer delegation |
| "praktikum", "lab" | Likely needs code + explanation sections |
| "review", "critical" | Needs critical analysis generation |
| "gambar", "diagram", "flowchart" | May need diagram-generator |
| "tabel", "table" | May need table-generator |

## Step 2: GENERATE — Build YAML Structure

### 2.1 Generate Task Name

If `task_name` not provided, generate from description:
- Extract key nouns: "review paper" → `review-paper`
- Lowercase, hyphenated: "Analisis Jurnal Ilmiah" → `analisis-jurnal-ilmiah`
- Max 30 chars

### 2.2 Build YAML

Assemble the complete YAML structure:

```yaml
# Auto-generated by template-creator
# Source: "{original_description}"
# Created: {date}

name: "{task_name}"
display_name: "{display_name}"
description: "{description_summary}"

document_class: "article"
paper_size: "a4paper"
font_size: "12pt"
language: "indonesian"

naming_convention: "{matkul}_{TaskName}_{nama}_{npm}"

cover:
  enabled: {true_if_cover_detected}
  template: "gunadarma-default"
  fields: [matkul, judul_tugas, nama, npm, kelas, dosen, tanggal]

sections:
  # ... generated sections ...

bibliography:
  enabled: {true_if_references_detected}
  style: "thebibliography"
```

### 2.3 Section Generation Rules

For each detected section, generate the YAML entry:

**`generate` type sections:**
```yaml
- id: "{normalized_id}"
  title: "{title}"
  type: "generate"
  required: true
  prompt_hint: >
    {auto_generated_prompt_based_on_section_name_and_context}
  min_words: {estimated_min}
  max_words: {estimated_max}
  latex_env: "section"
```

**Word count estimation:**

| Section Type | Min Words | Max Words |
|-------------|-----------|-----------|
| Latar belakang / Pendahuluan | 150 | 400 |
| Ringkasan / Summary | 100 | 300 |
| Metode | 150 | 400 |
| Pembahasan / Analisis | 200 | 800 |
| Critical Review | 300 | 800 |
| Kesimpulan | 100 | 300 |
| Saran | 50 | 200 |

**`auto` type sections:**
```yaml
- id: "identitas_paper"
  title: "Identitas Paper"
  type: "auto"
  source: "journal_data"
  fields: [title, authors, journal, year, doi]
  format: "table"
```

**`code_listing` type sections:**
```yaml
- id: "source_code"
  title: "Source Code"
  type: "code_listing"
  source: "{extra.code_path}"
  language: "java"
  line_numbers: true
  max_lines: 100
```

**`reference` type sections:**
```yaml
- id: "daftar_pustaka"
  title: "Daftar Pustaka"
  type: "reference"
  count: 5
  topic: "{task.topic}"
  max_age_years: 5
  format: "thebibliography"
```

### 2.4 Generate Prompt Hints

For `generate` type sections, create contextual prompt hints:

```
Section: "Latar Belakang"
→ prompt_hint: "Write background context explaining why {topic} is important
   in the field of {matkul}. Include current trends and challenges.
   Write in formal Indonesian academic style."

Section: "Pembahasan"
→ prompt_hint: "Provide detailed analysis and discussion about {topic}.
   Reference the methodology and results from previous sections.
   Write in formal Indonesian academic style."

Section: "Kesimpulan"
→ prompt_hint: "Summarize the key findings and insights from the analysis.
   State the main takeaways and their implications for {matkul}.
   Write in formal Indonesian academic style."
```

---

## Step 3: VALIDATE — Check YAML Correctness

### 3.1 Schema Validation

Verify the generated YAML against required structure:

- [ ] `name` is present and valid (lowercase, hyphenated)
- [ ] `sections` array is non-empty
- [ ] Each section has `id`, `title`, `type`
- [ ] Section `type` is one of: `auto`, `generate`, `user_input`, `code_listing`, `reference`, `delegate`
- [ ] `generate` sections have `prompt_hint`
- [ ] `code_listing` sections have `source` and `language`
- [ ] `reference` sections have `count` and `topic`
- [ ] No duplicate section `id`s
- [ ] `naming_convention` contains at least `{nama}` or `{npm}`

### 3.2 Logical Validation

- [ ] If bibliography enabled → at least one `reference` type section exists
- [ ] If `code_listing` section exists → `language` is specified
- [ ] Section order makes sense (cover first, bibliography last)
- [ ] No orphan `context_refs` (referencing non-existent section IDs)

---

## Step 4: CONFIRM — Present to User

Show the parsed structure for confirmation:

```
Template yang akan dibuat: review-paper.yaml

📄 Format Dokumen:
   Kelas: article | Ukuran: A4 | Font: 12pt | Bahasa: Indonesian

📋 Struktur Sections:
   1. [cover]           — Cover page (auto)
   2. [identitas_paper] — Identitas Paper (auto dari metadata)
   3. [ringkasan]       — Ringkasan (AI generate, 100-300 kata)
   4. [critical_review]  — Critical Review (AI generate, 300-800 kata)
   5. [kesimpulan]      — Kesimpulan (AI generate, 100-300 kata)
   6. [daftar_pustaka]  — Daftar Pustaka (5 referensi)

📚 Bibliography: Enabled (thebibliography)
📝 Naming: {matkul}_Review-Paper_{nama}_{npm}

Simpan template ini? [Y/n/edit]
```

If user says "edit" → allow inline modifications before saving.

---

## Step 5: SAVE — Write YAML File

### 5.1 Ensure Directory Exists

```powershell
New-Item -ItemType Directory -Force -Path "templates/task-types/custom"
```

### 5.2 Write File

Save to `templates/task-types/custom/{task_name}.yaml`

### 5.3 Confirm

```
✓ Template saved: templates/task-types/custom/review-paper.yaml
  
  Untuk menggunakan:
  "Buatkan tugas review-paper untuk matkul {matkul} tentang {topic}"
```

# Example Interactions

## Example 1: Simple format

**User:** "tugas review paper, formatnya: cover, identitas paper, ringkasan, critical review, kesimpulan, daftar pustaka"

**Result:** `custom/review-paper.yaml` with 5 sections + cover + bibliography

## Example 2: Praktikum report

**User:** "laporan praktikum, ada cover, tujuan, dasar teori, langkah percobaan, source code, penjelasan logika, hasil output, kesimpulan"

**Result:** `custom/laporan-praktikum.yaml` with:
- cover (auto)
- tujuan (generate)
- dasar_teori (generate)
- langkah_percobaan (generate)
- source_code (code_listing)
- logika (delegate → code-analyzer)
- hasil_output (user_input — screenshot/output)
- kesimpulan (generate)

## Example 3: Essay

**User:** "tugas essay 1000 kata tentang topik bebas, ga perlu cover, cuma judul sama isi aja"

**Result:** `custom/essay.yaml` with:
- cover disabled
- judul (auto from topic)
- isi (generate, min 800 max 1200 words)
- No bibliography

## Example 4: From existing file

**User:** "bikin template dari file tugas lama ini: C:\tugas\contoh.docx"

**Process:**
1. Read the docx file structure
2. Identify sections from headings
3. Generate YAML matching the structure
4. Confirm with user

# Error Handling

| Error | Action |
|-------|--------|
| Description too vague ("bikin tugas") | Ask: "Format tugasnya apa aja? Sebutin section-section yang dibutuhkan." |
| Conflicting requirements | Ask for clarification |
| Task name already exists | Ask: "Template '{name}' sudah ada. Overwrite? [Y/n]" |
| Invalid section type detected | Default to `generate` with warning |
